# LBT Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Listen-Before-Talk to chirpmunk-trx so TX is gated by a CAD-driven channel-busy flag; NACK with `error: "channel_busy"` on deadline.

**Architecture:** New `chirpmunk-blocks::cad::ChannelActivityDetector` block ported (minimal subset) from gr4-lora `ChannelActivityDetector.hpp`. A `StreamDuplicator<2>` is inserted upstream of `frame_sync` (both loopback and hardware paths) — one tap → existing decoder, second tap → CAD. CAD writes `Arc<AtomicBool>`. `dispatch_lora_tx` polls the flag with deadline; NACKs on timeout.

**Tech Stack:** Rust 2024, FutureSDR runtime, `rustfft` (already a chirpmunk-phy dep), `tokio` for async polls, `serde` for config.

## Self-contained chirp utility (inside cad.rs)

We do NOT add to `chirpmunk-phy::utils`. Inlining a 30-line reference-chirp generator into `cad.rs` avoids cross-crate API churn and keeps the LBT change isolated. The math is small and unambiguous (Vangelista & Calvagno notation) — no risk of drift from chirpmunk-phy's own chirps because both derive from the same closed-form formula.

```rust
// SPDX-License-Identifier: GPL-3.0-only

//! Listen-Before-Talk channel activity detector. Port (minimal subset)
//! of gr4-lora `blocks/include/gnuradio-4.0/lora/ChannelActivityDetector.hpp`.
//!
//! Algorithm (Vangelista & Calvagno 2024, Algorithm 1, single-SF, AND mode):
//!   1. Buffer 2 * sf_size * os_factor IQ samples (= a 2-symbol oversampled
//!      window).
//!   2. For each of the two `sf_size * os_factor` symbols:
//!        a. For each of `os_factor` sub-chip timing offsets:
//!            - decimate the symbol by `os_factor` (every nth sample
//!              starting at the offset);
//!            - element-wise multiply by `conj(upchirp_1x)` (= downchirp
//!              ref) for upchirp detection, or by `upchirp_1x` for
//!              downchirp detection;
//!            - FFT over `sf_size` bins;
//!            - peak_ratio = max|Y| / mean|Y|.
//!        b. Keep the maximum peak_ratio across offsets.
//!   3. Channel busy iff BOTH symbols' max ratios > alpha (AND mode).
//!   4. Hysteresis: release after `release_symbols` consecutive
//!      clear windows.

#![allow(dead_code)]

use std::sync::Arc;
use std::sync::atomic::{AtomicBool, Ordering};

use futuresdr::num_complex::Complex32;
use rustfft::{Fft, FftPlanner};

const TWO_PI: f32 = std::f32::consts::TAU;

/// Build a single-rate up-chirp of length `1 << sf` with bandwidth = sf_size
/// (i.e. os_factor=1). Phase model: `2π * (k²/(2N) − k/2)` per
/// gr4-lora `algorithm/utilities.hpp::build_ref_chirps`.
fn build_upchirp_1x(sf: u8) -> Vec<Complex32> {
    let n: usize = 1 << sf;
    let n_f = n as f32;
    (0..n)
        .map(|k| {
            let kf = k as f32;
            let phase = TWO_PI * (kf * kf / (2.0 * n_f) - kf / 2.0);
            Complex32::new(phase.cos(), phase.sin())
        })
        .collect()
}

/// Vangelista & Calvagno alpha: α = sqrt(−(4/π)·ln(1−(1−p_fa)^{1/L}))
/// where L = os_factor × M, M = 2^sf, p_fa = 0.001.
pub fn default_alpha(sf: u8, os_factor: u32) -> f32 {
    let m = (1u32 << sf) as f64;
    let l = f64::from(os_factor) * m;
    let p_fa = 0.001_f64;
    let inner = 1.0 - (1.0_f64 - p_fa).powf(1.0 / l);
    f32::from(((-(4.0 / std::f64::consts::PI) * inner.ln()).sqrt()) as f32)
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn upchirp_unit_magnitude() {
        let chirp = build_upchirp_1x(7);
        assert_eq!(chirp.len(), 128);
        for c in &chirp {
            let mag = (c.re * c.re + c.im * c.im).sqrt();
            assert!((mag - 1.0).abs() < 1e-5, "chirp must be unit-magnitude");
        }
    }

    #[test]
    fn alpha_grows_with_sf() {
        let a7 = default_alpha(7, 1);
        let a12 = default_alpha(12, 1);
        assert!(a7 > 0.0);
        assert!(a12 > a7, "higher SF -> more bins -> higher alpha");
    }

    #[test]
    fn alpha_grows_with_os() {
        let a_os1 = default_alpha(7, 1);
        let a_os4 = default_alpha(7, 4);
        assert!(a_os4 > a_os1);
    }
}
```

```rust
    /// (the simplest model: each 1x sample is repeated `os_factor` times).
    /// This is *not* the true polyphase chirp at os_factor>1, but it is
    /// dechirped correctly by the same conj(upchirp_1x) at any sub-chip
    /// offset (because every offset gives the same 1x sequence). Good
    /// enough for synthetic detector tests.

        let one = build_upchirp_1x(sf);
        let mut out = Vec::with_capacity(one.len() * os as usize);
        for &s in &one {
            for _ in 0..os {
                out.push(s);
            }
        }

    }
```

```rust
#[derive(Debug, Clone, Copy, Default)]
pub struct DetectResult {
    pub peak_ratio_up: f32,
    pub up_detected: bool,
}

pub struct Detector {
    sf: u8,
    os_factor: u32,
    sf_size: usize, // 1 << sf
    sym_len: usize, // sf_size * os_factor
    upchirp: Vec<Complex32>,
    downchirp: Vec<Complex32>, // = conj(upchirp); ref for detecting upchirp
    fft: Arc<dyn Fft<f32>>,
    scratch: Vec<Complex32>,
    fft_out: Vec<Complex32>,
    alpha: f32,
}

impl Detector {
    pub fn new(sf: u8, os_factor: u32) -> Self {
        let sf_size = 1usize << sf;
        let sym_len = sf_size * os_factor as usize;
        let upchirp = build_upchirp_1x(sf);
        let downchirp: Vec<Complex32> = upchirp.iter().map(|c| c.conj()).collect();
        let mut planner = FftPlanner::new();
        let fft = planner.plan_fft_forward(sf_size);
        let alpha = default_alpha(sf, os_factor);
        Self {
            sf,
            os_factor,
            sf_size,
            sym_len,
            upchirp,
            downchirp,
            fft,
            scratch: vec![Complex32::default(); sf_size],
            fft_out: vec![Complex32::default(); sf_size],
            alpha,
        }
    }

    pub fn alpha(&self) -> f32 {
        self.alpha
    }
    pub fn set_alpha(&mut self, alpha: f32) {
        self.alpha = alpha;
    }

    /// Sweep over `os_factor` sub-chip offsets, return the best peak ratio
    /// for one symbol's worth of oversampled samples (length `self.sym_len`).
    fn peak_ratio(&mut self, sym: &[Complex32], ref_chirp: &[Complex32]) -> f32 {
        debug_assert_eq!(sym.len(), self.sym_len);
        let mut best = 0.0_f32;
        for offset in 0..self.os_factor as usize {
            for k in 0..self.sf_size {
                self.scratch[k] = sym[offset + k * self.os_factor as usize] * ref_chirp[k];
            }
            self.fft_out.copy_from_slice(&self.scratch);
            self.fft.process(&mut self.fft_out);
            let mut peak = 0.0_f32;
            let mut total = 0.0_f32;
            for &y in &self.fft_out {
                let mag = (y.re * y.re + y.im * y.im).sqrt();
                total += mag;
                if mag > peak {
                    peak = mag;
                }
            }
            let mean = total / self.sf_size as f32;
            let ratio = if mean > 0.0 { peak / mean } else { 0.0 };
            if ratio > best {
                best = ratio;
            }
        }
        best
    }

    /// Process a 2-symbol oversampled window. AND mode: both symbols must
    /// exceed alpha for `up_detected = true`.
    pub fn process_window(&mut self, win: &[Complex32]) -> DetectResult {
        let need = self.sym_len * 2;
        debug_assert!(win.len() >= need, "window too short");
        let s1 = &win[..self.sym_len];
        let s2 = &win[self.sym_len..need];
        let ref_chirp = self.downchirp.clone();
        let r1 = self.peak_ratio(s1, &ref_chirp);
        let r2 = self.peak_ratio(s2, &ref_chirp);
        let max = r1.max(r2);
        let up_detected = (r1 > self.alpha) && (r2 > self.alpha);
        DetectResult {
            peak_ratio_up: max,
            up_detected,
        }
    }
}

/// Stateful streaming wrapper: accumulates input, emits busy/clear, applies
/// release-hysteresis. Writes through to an `Arc<AtomicBool>` so the TX
/// dispatch loop can poll it.
pub struct StreamingDetector {
    detector: Detector,
    win_len: usize,
    accum: Vec<Complex32>,
    fill: usize,
    release_symbols: u8,
    clean_count: u8,
    busy: Arc<AtomicBool>,
}

impl StreamingDetector {
    pub fn new(sf: u8, os_factor: u32, release_symbols: u8, busy: Arc<AtomicBool>) -> Self {
        let detector = Detector::new(sf, os_factor);
        let win_len = detector.sym_len * 2;
        Self {
            accum: vec![Complex32::default(); win_len],
            fill: 0,
            detector,
            win_len,
            release_symbols,
            clean_count: 0,
            busy,
        }
    }

    /// Feed any number of samples. Each time the accumulator fills to
    /// `win_len`, run a detection and update the busy flag.
    pub fn feed(&mut self, mut samples: &[Complex32]) {
        while !samples.is_empty() {
            let need = self.win_len - self.fill;
            let take = need.min(samples.len());
            self.accum[self.fill..self.fill + take].copy_from_slice(&samples[..take]);
            self.fill += take;
            samples = &samples[take..];
            if self.fill == self.win_len {
                let r = self.detector.process_window(&self.accum);
                if r.up_detected {
                    self.clean_count = 0;
                    self.busy.store(true, Ordering::Release);
                } else {
                    self.clean_count = self.clean_count.saturating_add(1);
                    if self.clean_count >= self.release_symbols {
                        self.busy.store(false, Ordering::Release);
                    }
                }
                self.fill = 0;
            }
        }
    }
}
```

```rust
// SPDX-License-Identifier: GPL-3.0-only

//! Dispatch a CBOR `lora_tx` request through a running Flowgraph's
//! Transmitter block. Honours `repeat` and `gap_ms`, drops the request
//! when `dry_run = true`. Optionally LBT-gates the dispatch on a
//! channel-busy flag. Returns the matching `lora_tx_ack`.

use std::sync::Arc;
use std::sync::atomic::{AtomicBool, Ordering};
use std::time::Duration;

use chirpmunk_cbor::{LoraTx, LoraTxAck};
use futuresdr::prelude::*;
use futuresdr::runtime::Timer;

#[derive(Debug, Clone, Copy)]
pub enum LbtOutcome {
    Clear,
    Timeout,
}

/// Poll `busy` until it reads `false` or `deadline_after` elapses.
/// Sleeps `poll_interval` between reads.
pub async fn wait_until_clear(
    busy: &AtomicBool,
    deadline_after: Duration,
    poll_interval: Duration,
) -> LbtOutcome {
    let start = std::time::Instant::now();
    loop {
        if !busy.load(Ordering::Acquire) {
            return LbtOutcome::Clear;
        }
        if start.elapsed() >= deadline_after {
            return LbtOutcome::Timeout;
        }
        Timer::after(poll_interval).await;
    }
}

/// LBT configuration for `dispatch_lora_tx`.
#[derive(Debug, Clone)]
pub struct LbtPolicy {
    pub busy: Arc<AtomicBool>,
    pub timeout: Duration,
    /// Defaults to 10 ms (matches gr4-lora `tx_worker.cpp`).
    pub poll_interval: Duration,
}

/// Send `req.payload` `req.repeat` times via the running Flowgraph's
/// transmitter `msg` message port. `gap_ms` separates repeats. When
/// `policy` is `Some`, each TX is gated on the channel-busy flag.
pub async fn dispatch_lora_tx(
    handle: &FlowgraphHandle,
    transmitter: BlockId,
    req: &LoraTx,
    policy: Option<&LbtPolicy>,
) -> LoraTxAck {
    let seq = req.seq.unwrap_or(0);
    let payload_len = req.payload.len();
    let repeat_req = req.repeat.unwrap_or(1).max(1);
    tracing::info!(
        seq,
        payload_len,
        repeat = repeat_req,
        dry_run = req.dry_run,
        lbt = policy.is_some(),
        "lora_tx dispatch entry"
    );
    if req.dry_run {
        return LoraTxAck::ok(seq);
    }
    if req.payload.is_empty() {
        return LoraTxAck::err(seq, "internal");
    }
    let gap = Duration::from_millis(req.gap_ms.unwrap_or(0) as u64);

    for i in 0..repeat_req {
        if let Some(policy) = policy {
            match wait_until_clear(&policy.busy, policy.timeout, policy.poll_interval).await {
                LbtOutcome::Clear => {}
                LbtOutcome::Timeout => {
                    tracing::warn!(seq, attempt = i + 1, of = repeat_req, "LBT timeout");
                    return LoraTxAck::err(seq, "channel_busy");
                }
            }
        }
        let pmt = Pmt::Blob(req.payload.clone());
        match handle.post(transmitter, "msg", pmt).await {
            Ok(()) => tracing::info!(
                seq,
                attempt = i + 1,
                of = repeat_req,
                "Pmt::Blob posted to Transmitter"
            ),
            Err(e) => {
                tracing::warn!(error = %e, seq, "tx dispatch post failed");
                return LoraTxAck::err(seq, "internal");
            }
        }
        if i + 1 < repeat_req && !gap.is_zero() {
            Timer::after(gap).await;
        }
    }
    LoraTxAck::ok(seq)
}
```

## Execution Handoff

Plan complete and saved to `docs/superpowers/plans/2026-05-08-lbt-plan.md`. Two execution options:

1. **Subagent-Driven** — fresh subagent per task, review between. Higher token cost, lower risk per turn.
2. **Inline Execution** — current session, batched checkpoints. Lower cost, holds more context.

For chirpmunk's tool-routing pattern (each bash/edit is a turn), inline is faster.
